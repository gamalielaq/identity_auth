# Contexto Identity API

Esta rama queda como base reusable de autenticación e identidad.

Objetivo: proveer registro, login, refresh token, logout, usuarios, aplicaciones cliente y sesiones sin depender de ningún dominio de negocio.

No contiene lógica de tareas, familias, miembros familiares, restaurante, productos ni turnos.

La base está pensada para poder extraerse luego a un servicio independiente `identity-api`.
